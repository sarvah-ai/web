Context Keeper ()
●Goal: To devise an optimal FetchPlan to acquire high-quality, relevant data and to maintain an immutable ProvenanceLedger for every piece of information. ●Enriched Responsibilities: ○Dynamic Source Prioritization: The FetchPlan is not static. The agent prioritizes sources based on the query's context. For a query on iconography, it will prioritize Iconclass and scholarly articles. For attribution, it will prioritize museum APIs and catalogues raisonnés. ○Query Formulation & Adaptation: It generates precise, structured queries (e.g., SPARQL for Wikidata, specific API calls for the MET) tailored to each source. It can also adapt queries on the fly if an initial search yields poor results. ○Comprehensive Provenance Tracking: The ProvenanceLedger is more than a list of URLs. It records the source's reliability score, the exact data snippet retrieved, any transformations applied (e.g., "normalized date from 'c. 1434' to '1434'"), and the data's license. This ensures full transparency and reproducibility. ○Orchestration of Parallel Fetches: To ensure speed, it orchestrates parallel data retrieval requests, managing dependencies (e.g., first get the Wikidata ID for an artwork, then use that ID to query other databases).
